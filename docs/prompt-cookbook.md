# 推荐 Prompt 手册

这份手册给共享使用者一个简单原则：别只说“帮我写报告”，而要把任务边界、目标篇幅、时间和优先动作说清楚。

## 标准咨询报告

```text
帮我写一份关于 XX 主题的咨询报告，目标读者是 XX，预计 XX 页，截止时间是 XX。先帮我确认大纲和核心判断，再开始写正文。
```

## 执行摘要

```text
基于现有报告内容，帮我提炼一版面向高层的执行摘要，控制在 1-2 页，突出关键发现、核心建议和下一步行动，不要写成正文说明。
```

## 形式交付 / 救火模式

```text
这是一个形式交付场景，时间很紧，先保证结构完整和篇幅达标。请按救火模式推进，先出大纲和主体章节，再统一做质量检查。
```

## 继续已有项目

```text
继续已有项目。请先读取 plan 里的项目概览和 progress，再基于当前上下文补写 XX 章节，不要重开新项目。
```

## 质量审查

```text
请对这份报告做质量审查，重点看元叙事、自我解释句、后台推进词、数据来源和章节厚度。
```

## 可审草稿

```text
正文已经基本完成。请先跑一遍质量检查，再帮我导出一版可审草稿，用于内部传阅和批注，不要表述成最终提交版。
```
