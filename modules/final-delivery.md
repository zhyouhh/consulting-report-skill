# 最终交付与可审草稿模块

## 一、定位

本模块用于把已经成形的 Markdown 正文整理为可流转、可批注、可预审的交付版本。它解决的是“先形成可看的文档”，不是“替代最终中文排版”。

## 二、适用场景

- 需要快速给领导、项目经理或客户内部做预审
- 需要把 Markdown 正文转成 `docx` 便于批注
- 需要形成一版接近终稿的可传阅材料

不适用场景：

- 对公司模板、页眉页脚、页码、图注格式有严格要求的最终正式交付
- 需要高度精细的中文排版和跨页图表调整

## 三、推荐流程

### 3.1 导出前检查

先确认以下内容：

- 标题层级已经基本稳定
- 图表和表格编号已检查
- 高风险占位符已清理
- 元叙事和后台推进词已基本清除

### 3.2 导出 `docx` 可审草稿

Windows PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/export_draft.ps1 -InputPath report.md -OutputDir output
```

macOS / Linux：

```bash
bash scripts/export_draft.sh report.md output
```

### 3.3 预览和抽查

导出后至少检查：

- 标题层级是否正常
- 目录和正文是否存在明显错位
- 表格是否溢出
- 关键图表说明是否可读

### 3.4 人工终排

在正式交付前，仍建议在 Word 或正式模板中完成：

- 公司模板套用
- 页眉页脚、页码
- 自动目录更新
- 图表位置与分页微调
- 字体、字号、段前段后统一

## 四、边界说明模板

如果用户要求直接出 Word，可明确说明：

> 当前导出的版本为可审草稿，适合内部评审、传阅和批注；最终中文排版、模板适配、页眉页脚和细节格式仍建议在正式模板中人工完成。

## 五、与质量检查的关系

建议顺序为：

1. 先运行 `quality_check`
2. 再执行导出
3. 导出后做一次人工抽查

不要在存在明显占位符、后台词或编号问题时直接导出，否则会把写作阶段的问题带进评审环节。
