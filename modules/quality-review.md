# 报告质量审查模块

## 一、审查目标

质量审查的目标不是把所有报告都套进同一个模板，而是确保报告在既定交付边界内做到三件事：内容可信、结构匹配、表达专业。对于短周期项目，审查更强调“先把高风险问题拦住”；对于正式终稿，则要进一步检查结构完整性和呈现一致性。

## 二、三级审查体系

### 2.1 一级审查：内容可信性

**审查重点**：
- 数据和事实是否可追溯
- 判断与证据是否一致
- 结论是否能够落到客户关心的问题

**检查清单**：
- [ ] 核心数据是否标注来源和时间
- [ ] 事实、口径、引用是否前后一致
- [ ] 重要判断是否有证据支撑
- [ ] 高风险观点是否明确区分“事实”与“研判”
- [ ] 建议是否能够对应前文分析结论

### 2.2 二级审查：结构匹配性

**审查重点**：
- 是否符合已确认的大纲和颗粒度
- 章节逻辑是否顺畅
- 重点内容是否展开到位

**检查清单**：
- [ ] 章节设置是否与已确认目录一致
- [ ] 一级、二级、三级标题是否匹配内容深度
- [ ] 关键章节是否存在明显缺口或失衡
- [ ] 可选章节是否按项目需要保留，例如研究概述、术语与定义、案例页或附加说明
- [ ] 各章之间是否形成“背景-分析-判断-举措”或其他约定逻辑链

### 2.3 三级审查：表达与呈现

**审查重点**：
- 语言是否像咨询交付，而不是学术论文或后台沟通记录
- 图表、编号、标题、目录是否整齐
- 文档是否适合进入评审或排版阶段

**检查清单**：
- [ ] 标题层级是否正确，且不存在硬拆三级标题
- [ ] 是否存在元叙事、自我解释句、后台推进词
- [ ] 图表编号、表格编号、目录层级是否连续
- [ ] 段落长度、列表密度、加粗使用是否均衡
- [ ] 若需要导出 `可审草稿`，是否已完成导出前复核

## 三、专项审查要点

### 3.1 数据质量审查

- 数据来源是否权威、可复核
- 数据时间是否满足当前报告场景
- 同一指标在不同章节是否存在口径漂移
- 结论是否从数据推导，而不是先有观点后拼素材

### 3.2 语言质量审查

- 是否存在机械过渡词和空洞强调句
- 是否出现“本章将”“后文会”“本报告不展开”等自我解释
- 是否泄漏后台推进词，例如“技术规范书”“内部材料”“AI reference”
- 是否有“看起来很专业但没有信息量”的套话

### 3.3 图表质量审查

- 图表是否真正承担信息表达，而不是重复正文
- 图表标题是否完整，时间、对象、指标是否说清楚
- 关键图表是否与正文洞察互相支撑
- 配色、标签、留白是否达到评审可读标准

### 3.4 可审草稿审查

当需要导出 Word 或 PDF 预审版本时，额外检查：

- 导出文件名是否清晰
- 目录、标题、表格在导出后是否基本可读
- 页内是否存在明显溢出、乱码、错位
- 是否已明确提醒“这是可审草稿，不是最终终排版”

## 四、常见风险分级

### 4.1 高风险

- 数据无来源或来源明显不可靠
- 报告中残留占位符，如 `XXX`、`TBD`、`待确认`
- 暴露后台推进信息，如“内部材料”“技术规范书”
- 重要结论无法对应前文分析

### 4.2 中风险

- 章节失衡，某些关键章节明显过薄
- 自我解释句过多，正文像写作说明书
- 列表过碎，造成篇幅很多但信息密度不足
- 图表与正文重复，没有新增洞察

### 4.3 低风险

- 机械过渡词零散出现
- 个别段落过长或过短
- 加粗、标点、空行等格式细节不统一
- 可选章节命名还可进一步贴近客户语境

## 五、审查流程

### 5.1 自审阶段

**执行人**：撰写人  
**动作**：
- 跑一次 `quality_check` 脚本
- 先清高风险，再收中风险
- 标记仍待核实的数据和来源

### 5.2 互审阶段

**执行人**：团队成员  
**动作**：
- 重点看逻辑链、章节厚度、建议可执行性
- 检查是否仍有明显 AI 味和后台词
- 判断是否适合进入评审会或客户版排版

### 5.3 终审阶段

**执行人**：项目负责人  
**动作**：
- 核对与任务目标、交付边界、大纲版本是否一致
- 确认关键结论、关键数据、关键图表
- 决定直接交付、继续精修，或导出 `可审草稿`

## 六、快速审查工具

```bash
# Linux/macOS
bash scripts/quality_check.sh report.md

# Windows PowerShell
powershell -ExecutionPolicy Bypass -File scripts/quality_check.ps1 -FilePath report.md
```

如需导出 `docx` 预审版本，可结合 `modules/final-delivery.md` 使用导出脚本。
