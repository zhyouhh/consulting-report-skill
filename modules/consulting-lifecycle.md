# 咨询项目生命周期模块

## 一、阶段总览

| 阶段 | 目标 | 关键活动 | 交付物 |
|------|------|----------|--------|
| S0 项目启动 | 明确范围和目标 | 需求访谈、范围界定 | project-overview.md |
| S1 研究设计 | 确定方法论 | 分析框架、数据源 | research-plan.md |
| S2 数据收集 | 获取信息 | 访谈、调研、数据 | data-log.md |
| S3 分析洞察 | 发现问题 | 数据分析、模式识别 | analysis-notes.md |
| S4 报告撰写 | 形成文档 | 写作、图表 | report-draft/ |
| S5 质量审查 | 确保质量 | 事实核查、逻辑验证 | review-checklist.md |
| S6 演示准备 | 准备汇报 | PPT、演讲稿 | presentation/ |
| S7 交付跟进 | 完成项目 | 交付、反馈 | delivery-log.md |

## 二、各阶段详细指南

### S0 项目启动
**AI 可执行动作**:
1. 创建 project-overview.md
2. 记录客户背景、项目目标、范围、时间线
3. 识别关键利益相关方
4. 明确成功标准

### S1 研究设计
**AI 可执行动作**:
1. 推荐分析框架（SWOT、Porter、PEST 等）
2. 规划数据来源（一手/二手）
3. 设计访谈大纲
4. 制定研究时间表

### S2 数据收集
**AI 可执行动作**:
1. 维护数据收集日志
2. 整理访谈记录
3. 汇总二手数据来源
4. 标注数据质量和可信度

### S3 分析洞察
**AI 可执行动作**:
1. 数据清洗和处理
2. 模式识别和趋势分析
3. 关键发现提炼
4. So What 测试

### S4 报告撰写
**AI 可执行动作**:
1. 按模板生成报告结构
2. 撰写各章节内容
3. 生成商业图表
4. 编写执行摘要

### S5 质量审查
**AI 可执行动作**:
1. 事实核查清单
2. 逻辑一致性检查
3. 数据来源验证
4. 语言质量审查

### S6 演示准备
**AI 可执行动作**:
1. 生成 PPT 大纲
2. 提炼核心信息
3. 准备 Q&A 预案
4. 设计视觉呈现

### S7 交付跟进
**AI 可执行动作**:
1. 整理最终交付物
2. 记录客户反馈
3. 归档项目文档
4. 总结经验教训
