# 常见错误清单（Common Gotchas）

> **重要**: Claude 会认真阅读此文档，避免重复犯错。每次写作前必须检查。

---

## 1. 写作常见错误

### 1.1 机械过渡词

❌ **错误示例**:
```
首先，分析市场现状。其次，评估竞争格局。最后，提出战略建议。
```

✅ **正确示例**:
```
市场规模达500亿元，年增长率15%。竞争格局呈现"一超多强"态势，
头部企业占据60%份额。基于此，建议采取差异化定位策略。
```

💡 **为什么这是错误**: 机械过渡词让报告像小学作文，缺乏专业性。

🔧 **如何避免**: 用数据、逻辑关系、因果链接替代过渡词。

---

### 1.2 空洞形容词

❌ **错误示例**:
```
该企业具有非常强大的竞争优势，市场表现极其优异，发展前景显著良好。
```

✅ **正确示例**:
```
该企业市占率35%（行业第一），毛利率42%（高于行业均值28%），
近三年营收复合增长率32%。
```

💡 **为什么这是错误**: 空洞形容词无法量化，缺乏说服力。

🔧 **如何避免**: 每个判断必须有数据支撑，用数字替代形容词。

---

### 1.3 缺少数据来源标注

❌ **错误示例**:
```
2025年中国新能源汽车销量达800万辆。
```

✅ **正确示例**:
```
2025年中国新能源汽车销量达800万辆（数据来源：中汽协，2026年1月）。
```

💡 **为什么这是错误**: 无法追溯数据来源，影响报告可信度。

🔧 **如何避免**: 每个数据必须标注来源和时间，格式：（数据来源：XX，YYYY年MM月）。

---

### 1.4 段落过长

❌ **错误示例**:
```
一个段落超过250字，包含市场分析、竞争格局、用户画像、技术趋势、
政策环境等多个主题，读者无法快速抓住重点，信息密度过高导致阅读疲劳...
```

✅ **正确示例**:
```
市场规模：2025年达500亿元，年增长率15%（数据来源：艾瑞咨询，2026年2月）。

竞争格局：CR5占比60%，呈现"一超多强"态势。头部企业A市占率25%，
B/C/D/E分别为10%/9%/8%/8%。
```

💡 **为什么这是错误**: 段落过长降低可读性，核心信息被淹没。

🔧 **如何避免**: 每段不超过250字，一段一个主题，用空行分隔。

---

### 1.5 列表格式不统一

❌ **错误示例**:
```
- 市场规模500亿元
- 增长率：15%
- 竞争格局呈现一超多强
- CR5=60%
```

✅ **正确示例**:
```
- 市场规模：500亿元（2025年）
- 年增长率：15%
- 竞争格局：一超多强，CR5占比60%
- 头部企业：A（25%）、B（10%）、C（9%）
```

💡 **为什么这是错误**: 格式不统一影响专业性，信息密度不一致。

🔧 **如何避免**: 统一格式为"指标名称：数值（单位/时间）"。

---

### 1.6 缺少 So What

❌ **错误示例**:
```
2025年市场规模500亿元，年增长率15%。
```

✅ **正确示例**:
```
2025年市场规模500亿元，年增长率15%。高增长意味着市场仍处于
快速扩张期，新进入者有机会通过差异化定位获取份额。
```

💡 **为什么这是错误**: 只陈述事实，未提炼洞察，客户无法理解数据意义。

🔧 **如何避免**: 每个数据后必须回答"So What"——这个数据说明什么问题？对决策有何启示？

---

### 1.7 结论与数据脱节

❌ **错误示例**:
```
数据显示市场增长率15%，竞争激烈。因此建议采取低价策略。
```

✅ **正确示例**:
```
数据显示市场增长率15%，处于快速扩张期，用户对价格敏感度较低
（价格弹性系数0.3）。因此建议采取高端定位策略，通过品质溢价获取利润。
```

💡 **为什么这是错误**: 结论未从数据推导，逻辑链断裂。

🔧 **如何避免**: 结论必须从数据推导，明确逻辑链：数据→洞察→结论。

---

### 1.8 过度使用被动语态

❌ **错误示例**:
```
市场被分析，竞争格局被评估，战略被制定。
```

✅ **正确示例**:
```
我们分析市场，评估竞争格局，制定战略。
```

💡 **为什么这是错误**: 被动语态削弱行动感，降低报告说服力。

🔧 **如何避免**: 优先使用主动语态，明确行动主体。

---

### 1.9 缺少量化标准

❌ **错误示例**:
```
建议提升客户满意度。
```

✅ **正确示例**:
```
建议在6个月内将NPS从35分提升至50分，客户复购率从40%提升至55%。
```

💡 **为什么这是错误**: 缺少量化标准，无法评估执行效果。

🔧 **如何避免**: 每个建议必须包含量化目标和时间节点。

---

### 1.10 图表与正文重复

❌ **错误示例**:
```
正文：2025年市场规模500亿元，年增长率15%。
图表：展示2025年市场规模500亿元，年增长率15%。
```

✅ **正确示例**:
```
正文：市场规模持续扩张，2025年达500亿元（图1）。高增长意味着
市场仍处于快速扩张期，新进入者有机会通过差异化定位获取份额。

图表：展示2020-2025年市场规模趋势，标注关键拐点。
```

💡 **为什么这是错误**: 图表与正文重复，浪费篇幅，未提炼洞察。

🔧 **如何避免**: 正文提炼洞察，图表展示数据，两者互补而非重复。

---

## 2. 结构常见错误

### 2.1 缺少执行摘要

❌ **错误示例**:
```
报告直接从"第一章 市场分析"开始。
```

✅ **正确示例**:
```
执行摘要（2-3页）
- 核心发现：市场规模500亿元，年增长率15%
- 关键洞察：竞争格局呈现"一超多强"，新进入者有机会
- 战略建议：采取差异化定位策略，聚焦细分市场
- 预期收益：3年内市占率达8%，营收10亿元
```

💡 **为什么这是错误**: 高层决策者只看执行摘要，缺少则无法快速理解报告核心。

🔧 **如何避免**: 每份报告必须包含执行摘要，篇幅2-3页，涵盖核心发现、洞察、建议。

---

### 2.2 章节逻辑不连贯

❌ **错误示例**:
```
第一章：市场分析
第二章：组织架构设计
第三章：竞争格局
第四章：财务预测
```

✅ **正确示例**:
```
第一章：市场分析（外部环境）
第二章：竞争格局（外部环境）
第三章：企业诊断（内部能力）
第四章：战略建议（基于内外部分析）
第五章：实施路径（落地方案）
```

💡 **为什么这是错误**: 章节顺序混乱，逻辑链断裂，读者无法理解分析框架。

🔧 **如何避免**: 遵循"外部→内部→战略→实施"逻辑，或使用经典框架（如麦肯锡金字塔原理）。

---

### 2.3 结论与数据不匹配

❌ **错误示例**:
```
数据：市场增长率15%，毛利率42%，用户满意度85%。
结论：企业面临严重危机，建议立即收缩业务。
```

✅ **正确示例**:
```
数据：市场增长率15%，毛利率42%（高于行业均值28%），用户满意度85%。
结论：企业处于良好发展态势，建议加大投入，扩大市场份额。
```

💡 **为什么这是错误**: 结论与数据矛盾，损害报告可信度。

🔧 **如何避免**: 结论必须从数据推导，确保逻辑一致性。

---

### 2.4 缺少行动建议

❌ **错误示例**:
```
报告结尾：综上所述，市场前景广阔，企业应抓住机遇。
```

✅ **正确示例**:
```
行动建议：
1. 短期（3个月）：完成产品升级，推出高端系列
2. 中期（6个月）：拓展华东市场，建立3个区域中心
3. 长期（12个月）：实现营收10亿元，市占率8%

关键里程碑：
- Q1：完成产品研发，启动市场测试
- Q2：正式上市，达成首批1000个客户
- Q3-Q4：规模化推广，实现盈亏平衡
```

💡 **为什么这是错误**: 缺少具体行动建议，客户无法执行。

🔧 **如何避免**: 每份报告必须包含行动建议，明确时间节点、责任人、量化目标。

---

### 2.5 目录层级过深

❌ **错误示例**:
```
1. 市场分析
  1.1 市场规模
    1.1.1 整体市场
      1.1.1.1 一线城市
        1.1.1.1.1 北京市场
          1.1.1.1.1.1 朝阳区
```

✅ **正确示例**:
```
1. 市场分析
  1.1 市场规模：500亿元（2025年）
  1.2 区域分布：一线城市占比60%
  1.3 增长趋势：年增长率15%
```

💡 **为什么这是错误**: 层级过深导致结构复杂，读者迷失。

🔧 **如何避免**: 目录最多3级，超过3级则合并或拆分章节。

---

## 3. 数据常见错误

### 3.1 引用过时数据

❌ **错误示例**:
```
2020年市场规模300亿元（数据来源：XX研究院，2021年报告）。
基于此预测2025年市场规模。
```

✅ **正确示例**:
```
2025年市场规模500亿元（数据来源：XX研究院，2026年2月）。
近三年复合增长率15%。
```

💡 **为什么这是错误**: 过时数据导致分析失真，影响决策准确性。

🔧 **如何避免**: 优先使用最近6个月内的数据，明确标注数据时间。

---

### 3.2 数据来源不可追溯

❌ **错误示例**:
```
据统计，市场规模500亿元。
```

✅ **正确示例**:
```
市场规模500亿元（数据来源：艾瑞咨询《2026年中国XX行业研究报告》，2026年2月）。
```

💡 **为什么这是错误**: 无法追溯数据来源，客户质疑数据真实性。

🔧 **如何避免**: 每个数据必须标注来源、报告名称、发布时间。

---

### 3.3 数据与结论矛盾

❌ **错误示例**:
```
数据：市场增长率15%，用户满意度85%，复购率60%。
结论：市场饱和，建议退出。
```

✅ **正确示例**:
```
数据：市场增长率15%，用户满意度85%，复购率60%。
结论：市场处于快速增长期，用户粘性强，建议加大投入。
```

💡 **为什么这是错误**: 数据与结论矛盾，逻辑链断裂。

🔧 **如何避免**: 结论必须从数据推导，确保逻辑一致性。

---

### 3.4 缺少时间标注

❌ **错误示例**:
```
市场规模500亿元。
```

✅ **正确示例**:
```
2025年市场规模500亿元（数据来源：XX研究院，2026年2月）。
```

💡 **为什么这是错误**: 缺少时间标注，无法判断数据时效性。

🔧 **如何避免**: 每个数据必须标注时间，格式：YYYY年数据（数据来源：XX，YYYY年MM月）。

---

### 3.5 数据精度不一致

❌ **错误示例**:
```
市场规模500亿元，增长率15.3782%，毛利率42%。
```

✅ **正确示例**:
```
市场规模500亿元，增长率15.4%，毛利率42%。
```

💡 **为什么这是错误**: 数据精度不一致，显得不专业。

🔧 **如何避免**: 统一精度标准，金额保留整数或1位小数，百分比保留1位小数。

---

### 3.6 缺少对比基准

❌ **错误示例**:
```
企业毛利率42%。
```

✅ **正确示例**:
```
企业毛利率42%，高于行业均值28%，处于行业领先水平。
```

💡 **为什么这是错误**: 缺少对比基准，无法判断数据好坏。

🔧 **如何避免**: 每个数据必须提供对比基准（行业均值、竞品、历史数据）。

---

## 4. 格式常见错误

### 4.1 图表编号不连续

❌ **错误示例**:
```
图1：市场规模趋势
图3：竞争格局
图5：用户画像
```

✅ **正确示例**:
```
图1：市场规模趋势
图2：竞争格局
图3：用户画像
```

💡 **为什么这是错误**: 编号不连续显得不专业，影响报告质量。

🔧 **如何避免**: 图表编号必须连续，从1开始递增。

---

### 4.2 表格格式不统一

❌ **错误示例**:
```
表1：使用三线表
表2：使用全框线表格
表3：无边框表格
```

✅ **正确示例**:
```
所有表格统一使用三线表格式（顶线、底线、表头线）。
```

💡 **为什么这是错误**: 格式不统一影响专业性。

🔧 **如何避免**: 统一使用三线表格式，或根据客户要求统一格式。

---

### 4.3 段落间距不一致

❌ **错误示例**:
```
段落1

段落2


段落3
段落4
```

✅ **正确示例**:
```
段落1

段落2

段落3

段落4
```

💡 **为什么这是错误**: 间距不一致影响阅读体验。

🔧 **如何避免**: 段落间统一使用一个空行分隔。

---

### 4.4 加粗过度使用

❌ **错误示例**:
```
**市场规模****500亿元**，**年增长率****15%**，**竞争格局****一超多强**。
```

✅ **正确示例**:
```
市场规模500亿元，年增长率15%，竞争格局一超多强。

**核心洞察**：高增长意味着市场仍处于快速扩张期。
```

💡 **为什么这是错误**: 过度加粗降低可读性，失去强调效果。

🔧 **如何避免**: 仅对核心结论、关键数据、章节标题使用加粗。

---

### 4.5 图表标题不规范

❌ **错误示例**:
```
图1：市场规模
```

✅ **正确示例**:
```
图1：2020-2025年中国XX行业市场规模及增长趋势
```

💡 **为什么这是错误**: 标题不完整，无法独立理解图表内容。

🔧 **如何避免**: 图表标题必须包含时间、地域、指标、单位。

---

## 5. 流程常见错误

### 5.1 跳过门禁机制

❌ **错误示例**:
```
用户："写一份战略报告"
Claude：直接开始写作，未讨论需求。
```

✅ **正确示例**:
```
用户："写一份战略报告"
Claude："在开始前，我需要确认几个关键信息：
1. 报告目标：战略规划/融资路演/内部决策？
2. 目标篇幅：15页/30页/50页？
3. 核心问题：市场进入/业务转型/并购整合？
4. 交付时间：本周/本月？"
```

💡 **为什么这是错误**: 跳过需求讨论导致返工，浪费时间。

🔧 **如何避免**: 必须先讨论需求，确认目标、篇幅、核心问题后再执行。

---

### 5.2 未确认篇幅要求

❌ **错误示例**:
```
用户："写一份报告"
Claude：写了80页报告，用户只需要15页。
```

✅ **正确示例**:
```
用户："写一份报告"
Claude："请确认目标篇幅：
- 简版（15-20页）：核心结论+关键数据
- 标准版（30-40页）：完整分析+详细建议
- 详版（50-80页）：深度研究+附录"
```

💡 **为什么这是错误**: 篇幅不匹配导致返工或浪费。

🔧 **如何避免**: 必须先确认篇幅要求，根据篇幅调整内容深度。

---

### 5.3 未创建 plan/ 目录

❌ **错误示例**:
```
直接开始写作，未创建 plan/ 目录和 progress.md。
```

✅ **正确示例**:
```
1. 创建 plan/ 目录
2. 创建 progress.md 跟踪进度
3. 创建 outline.md 确认大纲
4. 开始写作
```

💡 **为什么这是错误**: 缺少进度跟踪，无法管理复杂项目。

🔧 **如何避免**: 每个项目必须创建 plan/ 目录，使用 progress.md 跟踪进度。

---

### 5.4 未更新 progress.md

❌ **错误示例**:
```
完成第一章后，未更新 progress.md，直接开始第二章。
```

✅ **正确示例**:
```
完成第一章后：
1. 更新 progress.md，标记第一章为"已完成"
2. 记录关键发现和待解决问题
3. 开始第二章
```

💡 **为什么这是错误**: 缺少进度记录，无法追溯工作历史。

🔧 **如何避免**: 每完成一个章节，必须更新 progress.md。

---

### 5.5 未进行质量检查

❌ **错误示例**:
```
完成报告后直接交付，未运行质量检查脚本。
```

✅ **正确示例**:
```
完成报告后：
1. 运行 scripts/quality_check.py
2. 检查数据来源、图表编号、So What
3. 修复问题后再交付
```

💡 **为什么这是错误**: 未检查导致低级错误流入交付物。

## 7. 快速参考

### 7.1 禁用词汇

- 机械过渡词：首先、其次、最后、综上所述
- 空洞形容词：非常、极其、显著、巨大、重要
- 模糊表述：大约、可能、也许、据说、有人认为

### 7.2 必用元素

- 数据来源：（数据来源：XX，YYYY年MM月）
- 量化标准：具体数字+单位+时间
- 对比基准：行业均值/竞品/历史数据
- So What：数据→洞察→结论

### 7.3 格式标准

- 段落间距：一个空行
- 图表编号：连续递增，从1开始
- 表格格式：统一使用三线表
- 加粗规则：仅用于核心结论、关键数据、章节标题
