# 执行摘要写作模块

## 一、执行摘要的作用

执行摘要是咨询报告最重要的部分，通常是决策者唯一会完整阅读的内容。

**目标读者**: CEO、高管、决策者
**阅读时间**: 2-3 分钟
**页数**: 1-2 页

## 二、标准结构

### 2.1 四段式结构

1. **背景与目标**（2-3 句）
   - 项目背景
   - 核心问题
   - 研究目标

2. **关键发现**（3-5 条）
   - 最重要的洞察
   - 用数据支撑
   - 每条 1-2 句

3. **核心建议**（3-5 条）
   - 可执行的建议
   - 标注优先级
   - 预期影响

4. **下一步行动**（2-3 句）
   - 立即行动项
   - 时间线
   - 资源需求

## 三、写作原则

1. **结论先行**: 第一句就说最重要的结论
2. **数据驱动**: 每个发现用数字支撑
3. **面向行动**: 建议具体可执行
4. **独立完整**: 不读正文也能理解

## 四、质量检查清单

- [ ] 是否在 1-2 页内
- [ ] 是否包含 3-5 个关键发现
- [ ] 每个发现是否有数据支撑
- [ ] 建议是否标注优先级
- [ ] 是否独立完整

## 五、完整示例

### 案例：某零售企业数字化转型咨询项目

```markdown
## 执行摘要

华联零售集团在电商冲击下市场份额持续下滑，本项目旨在诊断核心问题并提出数字化转型路径，帮助企业重获竞争优势。

**关键发现**：
1. **客户流失严重**：过去两年线下客流量下降37%，会员活跃度仅18%，远低于行业平均水平32%
2. **运营效率低下**：库存周转天数达89天（行业平均45天），缺货率高达23%，导致年损失约1.2亿元
3. **数字化能力薄弱**：IT投入占营收比例仅0.8%（行业平均2.5%），线上线下数据未打通，决策依赖经验而非数据
4. **组织能力不足**：数字化人才占比不足5%，中层管理者对新技术接受度低，变革阻力大

**核心建议**：
1. **高优先级**：构建全渠道会员体系 - 整合线上线下数据，实现精准营销，预期12个月内会员活跃度提升至35%，复购率提升20%
2. **高优先级**：部署智能供应链系统 - 引入需求预测和自动补货，预期库存周转天数降至55天，缺货率降至8%以下，年节省成本8000万元
3. **中优先级**：建立数据中台 - 打通各业务系统，支撑数据驱动决策，预期6个月内实现核心业务数据实时可视化
4. **中优先级**：推进组织转型 - 设立数字化部门，引进20名专业人才，开展全员数字化培训

**下一步行动**：建议在未来3个月内启动会员体系和供应链系统试点项目，首期投入约2500万元，预计18个月收回投资。同时立即启动数字化人才招聘和组织架构调整。
```
